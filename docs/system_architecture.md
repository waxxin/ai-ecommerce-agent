# 系统架构说明

## 1. 项目概述

AI电商内容生成系统是一个基于LLM的企业级电商运营Agent系统，旨在帮助电商运营人员快速生成高质量的商品标题、卖点和多平台文案。

## 3. 核心数据流

### 3.1 内容生成流程

```
用户输入商品信息
    │
    ▼
┌─────────────────────────────────────────────────┐
│              AgentService.run()                 │
│                                                 │
│  Step 1: Plan (任务拆解)                        │
│  ┌─────────────────────────────────────────┐    │
│  │ - 分析用户输入                           │    │
│  │ - 确定任务类型                           │    │
│  │ - 生成执行步骤:                          │    │
│  │   ① 标题生成                            │    │
│  │   ② 卖点提取                            │    │
│  │   ③ FAQ生成                             │    │
│  └─────────────────────────────────────────┘    │
│                       │                         │
│                       ▼                         │
│  Step 2: Decide (是否需要RAG)                  │
│  ┌─────────────────────────────────────────┐    │
│  │ - 查询知识库获取平台规则                   │    │
│  │ - 标题规范、类目规则                      │    │
│  └─────────────────────────────────────────┘    │
│                       │                         │
│                       ▼                         │
│  Step 3: Execute (执行步骤)                   │
│  ┌─────────────────────────────────────────┐    │
│  │ - 调用LLMService.call_structured()       │    │
│  │ - 每个步骤生成结构化JSON输出               │    │
│  │ - 记录Token消耗和延迟                      │    │
│  └─────────────────────────────────────────┘    │
│                       │                         │
│                       ▼                         │
│  Step 4: Merge (合并结果)                    │
│  ┌─────────────────────────────────────────┐    │
│  │ - 统一输出结构:                          │    │
│  │   { titles, selling_points, faq }        │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
                       │
                       ▼
           ┌───────────────────┐
           │ HistoryService    │
           │ .create()         │
           │ (保存历史记录)     │
           └───────────────────┘
                       │
                       ▼
                 返回前端展示
```

## 4. 模块职责说明

| 模块 | 职责 | 关键类/函数 |
|------|------|-------------|
| **api/** | REST API接口层 | `/api/v1/generate`, `/api/v1/history` |
| **services/llm_service.py** | 统一LLM调用层 | `LLMService.call()`, `LLMService.call_structured()` |
| **services/agent_service.py** | Agent任务拆解引擎 | `EcommerceAgent.run()` |
| **services/prompt_service.py** | Prompt模板管理 | `PromptService.render_prompt()` |
| **services/rag_service.py** | RAG知识检索 | `RAGService.retrieve()` |
| **services/history_service.py** | 历史记录管理 | `HistoryService.create_record()` |
| **models/** | 数据库模型 | SQLAlchemy ORM |
| **core/** | 配置与基础能力 | Settings, Database |

## 8. 安全设计

- **API密钥管理**: 使用环境变量存储，不在代码中硬编码
- **请求限制**: 可配置请求频率限制
- **数据加密**: 数据库密码使用环境变量
- **CORS**: 配置允许的源地址
